# NFR Requirements - Unit 2: Menu

## 1. 성능 요구사항

| 항목 | 요구사항 | 근거 |
|---|---|---|
| 메뉴 조회 응답 시간 | ≤ 500ms | 고객 태블릿 UX, 매장당 메뉴 수 제한적 |
| 이미지 업로드 응답 시간 | ≤ 3s (5MB 기준) | 관리자 메뉴 등록 시 대기 허용 범위 |
| 메뉴 CRUD 응답 시간 | ≤ 300ms | 관리자 작업 효율 |
| 캐싱 | 없음 (MVP) | 매장당 메뉴 수십~수백 개, DB 직접 조회로 충분 |
| 페이지네이션 | 없음 | 전체 조회로 충분한 데이터 규모 |

## 2. 보안 요구사항

| 항목 | 요구사항 |
|---|---|
| 인증 | JWT 기반 (Unit 1 AuthMiddleware 활용) |
| 인가 | 역할 기반 — tablet: 조회만, store_admin: CRUD |
| 멀티테넌시 | 모든 쿼리에 store_id 필터 필수 |
| 파일 업로드 | Content-Type 검증, 파일 크기 제한 (5MB), UUID 파일명 |
| 입력 검증 | Pydantic 스키마로 모든 입력 검증 |

## 3. 확장성 요구사항

| 항목 | 현재 (MVP) | 향후 확장 |
|---|---|---|
| 이미지 저장 | 로컬 파일시스템 | AWS S3 마이그레이션 |
| 캐싱 | 없음 | 인메모리 캐싱 (TTL) |
| 이미지 처리 | 원본 저장 | 리사이징/썸네일 |
| 검색 | 카테고리별 필터 | 키워드 검색 |

## 4. 가용성 요구사항

| 항목 | 요구사항 |
|---|---|
| 메뉴 조회 | 매장 운영 시간 중 가용 (단일 서버) |
| 이미지 서빙 | FastAPI StaticFiles로 직접 서빙 |
| 장애 대응 | 이미지 로드 실패 시 기본 이미지 표시 (프론트엔드) |

## 5. 유지보수성 요구사항

| 항목 | 요구사항 |
|---|---|
| 코드 구조 | 3-Layer (Router → Service → Repository) |
| FileService | 인터페이스 분리 (추후 S3 교체 용이) |
| 테스트 | Repository, Service, Router 각 계층 단위 테스트 |
| 타입 안전성 | Pydantic 스키마 + TypeScript 타입 |

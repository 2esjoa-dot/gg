### Epic EP-C01: 테이블 인증 및 세션

#### US-C01: 태블릿 초기 설정 후 자동 로그인
**As a** 고객 (매장 방문자)
**I want** 태블릿에 별도 로그인 없이 바로 메뉴 화면을 볼 수 있기를
**So that** 복잡한 절차 없이 즉시 주문을 시작할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 관리자가 태블릿 초기 설정을 완료한 상태 **When** 태블릿 브라우저를 열면 **Then** 저장된 인증 정보로 자동 로그인되어 메뉴 화면이 표시된다
2. **Given** 자동 로그인 성공 **When** 메뉴 화면이 로드되면 **Then** 해당 매장의 메뉴 데이터가 정상 표시된다
3. **Given** 저장된 인증 정보가 없는 상태 **When** 태블릿 브라우저를 열면 **Then** 초기 설정 화면(매장 식별자, 테이블 번호, 비밀번호 입력)이 표시된다
4. **Given** 자동 로그인 실패 (잘못된 인증 정보) **When** 로그인 시도 시 **Then** 에러 메시지를 표시하고 초기 설정 화면으로 이동한다
5. **Given** 브라우저 새로고침 **When** 페이지가 리로드되면 **Then** 로컬 저장된 인증 정보로 재로그인되어 메뉴 화면이 유지된다

**Dependencies**: EP-A03 (관리자가 테이블 초기 설정 완료 필요)

#### US-C02: 테이블 세션 자동 시작
**As a** 고객
**I want** 첫 주문 시 자동으로 새 세션이 시작되기를
**So that** 이전 손님의 주문 내역과 분리된 나만의 주문 이력을 가질 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 활성 세션이 없는 테이블 **When** 고객이 첫 주문을 전송하면 **Then** 새로운 세션 ID가 생성되고 주문에 연결된다
2. **Given** 이미 활성 세션이 있는 테이블 **When** 추가 주문을 전송하면 **Then** 기존 세션 ID에 주문이 추가된다
3. **Given** 관리자가 이용 완료 처리한 테이블 **When** 다음 고객이 첫 주문을 전송하면 **Then** 새로운 세션 ID가 생성된다
4. **Given** 세션 시작 **When** 세션이 생성되면 **Then** 16시간 유효 기간이 설정된다
5. **Given** 세션 유효 기간 만료 **When** 주문을 시도하면 **Then** 새 세션이 자동 생성된다

**Dependencies**: US-C04 (주문 생성 기능 필요)

### Epic EP-C02: 메뉴 탐색

#### US-C03: 카테고리별 메뉴 조회
**As a** 고객
**I want** 카테고리별로 분류된 메뉴를 볼 수 있기를
**So that** 원하는 종류의 음식을 빠르게 찾을 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 메뉴 화면 진입 **When** 페이지가 로드되면 **Then** 카테고리 목록과 첫 번째 카테고리의 메뉴가 표시된다
2. **Given** 카테고리 목록 표시 **When** 특정 카테고리를 터치하면 **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다
3. **Given** 메뉴 목록 **When** 각 메뉴 카드를 보면 **Then** 메뉴명, 가격, 이미지가 카드 형태로 표시된다
4. **Given** 메뉴 카드 **When** 카드를 터치하면 **Then** 메뉴 상세 정보(설명, 큰 이미지)가 표시된다
5. **Given** 터치 인터페이스 **When** 버튼/카드를 확인하면 **Then** 최소 44x44px 터치 타겟 크기를 만족한다
6. **Given** 관리자가 메뉴를 비활성화한 경우 **When** 고객이 메뉴를 조회하면 **Then** 비활성화된 메뉴는 표시되지 않는다

**Dependencies**: EP-A04 (메뉴 데이터 등록 필요)

### Epic EP-C03: 장바구니

#### US-C04: 장바구니에 메뉴 추가 및 관리
**As a** 고객
**I want** 원하는 메뉴를 장바구니에 담고 수량을 조절할 수 있기를
**So that** 한 번에 여러 메뉴를 주문할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 메뉴 카드/상세 화면 **When** "담기" 버튼을 터치하면 **Then** 해당 메뉴가 장바구니에 수량 1로 추가된다
2. **Given** 장바구니에 동일 메뉴가 있는 상태 **When** 같은 메뉴를 다시 담으면 **Then** 수량이 1 증가한다
3. **Given** 장바구니 화면 **When** 수량 증가/감소 버튼을 터치하면 **Then** 수량이 변경되고 총 금액이 즉시 재계산된다
4. **Given** 수량이 1인 메뉴 **When** 감소 버튼을 터치하면 **Then** 해당 메뉴가 장바구니에서 삭제된다
5. **Given** 장바구니에 메뉴가 있는 상태 **When** "비우기" 버튼을 터치하면 **Then** 장바구니가 완전히 비워진다
6. **Given** 장바구니에 메뉴가 있는 상태 **When** 브라우저를 새로고침하면 **Then** 장바구니 내용이 로컬 스토리지에서 복원된다
7. **Given** 장바구니 **When** 메뉴가 추가/삭제/수량 변경되면 **Then** 총 금액이 실시간으로 업데이트된다

**Dependencies**: US-C03 (메뉴 조회 기능 필요)

### Epic EP-C04: 주문

#### US-C05: 주문 생성 및 확인
**As a** 고객
**I want** 장바구니의 메뉴를 주문으로 확정할 수 있기를
**So that** 매장에 내 주문이 전달된다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 장바구니에 1개 이상 메뉴가 있는 상태 **When** "주문하기" 버튼을 터치하면 **Then** 주문 내역 최종 확인 화면이 표시된다
2. **Given** 주문 확인 화면 **When** "주문 확정" 버튼을 터치하면 **Then** 서버에 주문이 전송된다 (매장ID, 테이블ID, 메뉴목록, 총금액, 세션ID 포함)
3. **Given** 주문 전송 성공 **When** 서버 응답을 받으면 **Then** 주문 번호가 표시되고, 장바구니가 비워지고, 5초 후 메뉴 화면으로 자동 리다이렉트된다
4. **Given** 주문 전송 실패 **When** 서버 에러 응답을 받으면 **Then** 에러 메시지가 표시되고 장바구니는 유지된다
5. **Given** 빈 장바구니 **When** "주문하기" 버튼을 확인하면 **Then** 버튼이 비활성화 상태이다
6. **Given** 활성 세션이 없는 테이블 **When** 첫 주문이 확정되면 **Then** 새 세션이 자동 생성되고 주문에 연결된다

**Dependencies**: US-C04 (장바구니 기능), US-C02 (세션 관리)

### Epic EP-C05: 주문 내역

#### US-C06: 현재 세션 주문 내역 조회
**As a** 고객
**I want** 현재 세션에서 주문한 내역과 상태를 확인할 수 있기를
**So that** 내 주문이 정상 접수되었는지, 현재 어떤 상태인지 알 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 주문 내역 화면 진입 **When** 페이지가 로드되면 **Then** 현재 세션의 주문만 시간 역순으로 표시된다
2. **Given** 주문 목록 **When** 각 주문을 확인하면 **Then** 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다
3. **Given** 주문 내역 화면 **When** 30초가 경과하면 **Then** 자동으로 서버에서 최신 상태를 폴링하여 업데이트한다
4. **Given** 이전 세션(이용 완료 처리된) 주문 **When** 주문 내역을 조회하면 **Then** 이전 세션 주문은 표시되지 않는다
5. **Given** 주문이 많은 경우 **When** 목록을 스크롤하면 **Then** 페이지네이션 또는 무한 스크롤로 추가 주문이 로드된다
6. **Given** 주문 상태가 변경된 경우 **When** 다음 폴링 시점에 **Then** 변경된 상태가 반영되어 표시된다

**Dependencies**: US-C05 (주문 생성 기능)

### Epic EP-A01: 매장 인증

#### US-A01: 관리자 로그인
**As a** 매장 관리자
**I want** 매장 식별자와 계정 정보로 로그인할 수 있기를
**So that** 내 매장의 주문과 메뉴를 관리할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 로그인 화면 **When** 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인 버튼을 클릭하면 **Then** 인증 성공 시 JWT 토큰이 발급되고 대시보드로 이동한다
2. **Given** 로그인 성공 **When** 세션이 시작되면 **Then** 16시간 동안 유효한 JWT 토큰이 로컬에 저장된다
3. **Given** 잘못된 인증 정보 **When** 로그인을 시도하면 **Then** "인증 정보가 올바르지 않습니다" 에러 메시지가 표시된다
4. **Given** 연속 5회 로그인 실패 **When** 추가 로그인을 시도하면 **Then** 15분간 로그인이 차단된다
5. **Given** 로그인된 상태 **When** 브라우저를 새로고침하면 **Then** 저장된 JWT 토큰으로 세션이 유지된다
6. **Given** 16시간 경과 **When** 토큰이 만료되면 **Then** 자동 로그아웃되고 로그인 화면으로 이동한다
7. **Given** 비밀번호 저장 **When** 서버에 저장될 때 **Then** bcrypt로 해싱되어 저장된다

**Dependencies**: EP-H01 (매장 등록 필요), EP-A05 (관리자 계정 등록 필요)

### Epic EP-A02: 실시간 주문 모니터링

#### US-A02: 실시간 주문 대시보드
**As a** 매장 관리자
**I want** 들어오는 주문을 실시간으로 대시보드에서 확인할 수 있기를
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 대시보드 화면 **When** 페이지가 로드되면 **Then** 테이블별 카드가 그리드 레이아웃으로 표시된다
2. **Given** SSE 연결 활성 **When** 고객이 새 주문을 생성하면 **Then** 2초 이내에 해당 테이블 카드에 주문이 표시된다
3. **Given** 테이블 카드 **When** 카드를 확인하면 **Then** 총 주문액과 최신 주문 미리보기가 표시된다
4. **Given** 신규 주문 도착 **When** 대시보드에 표시될 때 **Then** 시각적 강조(색상 변경/애니메이션)로 구분된다
5. **Given** 테이블 카드 **When** 클릭하면 **Then** 해당 테이블의 전체 주문 메뉴 목록 상세가 표시된다
6. **Given** SSE 연결 끊김 **When** 네트워크 문제 발생 시 **Then** 자동 재연결을 시도하고 연결 상태를 표시한다
7. **Given** 대시보드 **When** 테이블 필터를 적용하면 **Then** 선택한 테이블만 표시된다

**Dependencies**: US-C05 (고객 주문 생성 기능)

#### US-A03: 주문 상태 변경
**As a** 매장 관리자
**I want** 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있기를
**So that** 주문 처리 진행 상황을 추적하고 고객에게 알릴 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 주문 상세 보기 **When** 상태 변경 버튼을 클릭하면 **Then** 다음 상태로 변경된다 (대기중→준비중→완료)
2. **Given** 상태 변경 성공 **When** 서버 응답을 받으면 **Then** 대시보드에 즉시 반영된다
3. **Given** 상태가 "완료"인 주문 **When** 상태 변경을 시도하면 **Then** 더 이상 변경할 수 없다 (버튼 비활성화)
4. **Given** 상태 변경 **When** 고객이 다음 폴링 시 조회하면 **Then** 변경된 상태가 반영되어 표시된다
5. **Given** 상태 변경 실패 **When** 서버 에러 발생 시 **Then** 에러 메시지를 표시하고 이전 상태를 유지한다

**Dependencies**: US-A02 (주문 대시보드)

### Epic EP-A03: 테이블 관리

#### US-A04: 테이블 초기 설정
**As a** 매장 관리자
**I want** 태블릿에 테이블 번호와 비밀번호를 설정할 수 있기를
**So that** 고객이 해당 태블릿에서 자동 로그인하여 주문할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 테이블 관리 화면 **When** "테이블 추가" 버튼을 클릭하면 **Then** 테이블 번호, 비밀번호 입력 폼이 표시된다
2. **Given** 입력 폼 **When** 테이블 번호와 비밀번호를 입력하고 저장하면 **Then** 테이블이 등록되고 16시간 세션이 생성된다
3. **Given** 이미 존재하는 테이블 번호 **When** 동일 번호로 등록을 시도하면 **Then** 중복 에러 메시지가 표시된다
4. **Given** 테이블 등록 성공 **When** 태블릿에서 해당 정보로 접속하면 **Then** 자동 로그인이 활성화된다
5. **Given** 비밀번호 입력 **When** 유효성 검사 시 **Then** 최소 4자 이상이어야 한다

**Dependencies**: US-A01 (관리자 로그인)

#### US-A05: 주문 삭제
**As a** 매장 관리자
**I want** 잘못된 주문을 삭제할 수 있기를
**So that** 고객의 실수나 시스템 오류로 인한 잘못된 주문을 정정할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 테이블 주문 상세 **When** 특정 주문의 "삭제" 버튼을 클릭하면 **Then** 확인 팝업이 표시된다
2. **Given** 확인 팝업 **When** "확인"을 클릭하면 **Then** 주문이 즉시 삭제된다
3. **Given** 주문 삭제 성공 **When** 대시보드가 업데이트되면 **Then** 해당 테이블의 총 주문액이 재계산된다
4. **Given** 확인 팝업 **When** "취소"를 클릭하면 **Then** 삭제가 취소되고 이전 상태로 돌아간다
5. **Given** 주문 삭제 실패 **When** 서버 에러 발생 시 **Then** 에러 메시지를 표시하고 주문을 유지한다

**Dependencies**: US-A02 (주문 대시보드)

#### US-A06: 테이블 이용 완료 (세션 종료)
**As a** 매장 관리자
**I want** 고객이 떠난 후 테이블 세션을 종료할 수 있기를
**So that** 다음 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다

**Priority**: Must Have

**Acceptance Criteria**:
1. **Given** 테이블 카드 **When** "이용 완료" 버튼을 클릭하면 **Then** 확인 팝업이 표시된다
2. **Given** 확인 팝업 **When** "확인"을 클릭하면 **Then** 현재 세션의 주문 내역이 과거 이력으로 이동된다
3. **Given** 이용 완료 처리 성공 **When** 대시보드가 업데이트되면 **Then** 해당 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
4. **Given** 이용 완료된 테이블 **When** 다음 고객이 첫 주문을 하면 **Then** 새 세션이 시작되고 이전 주문은 보이지 않는다
5. **Given** 이용 완료 처리 **When** 과거 이력에 저장될 때 **Then** 세션 ID, 완료 시각이 함께 기록된다
6. **Given** 확인 팝업 **When** "취소"를 클릭하면 **Then** 세션이 유지된다

**Dependencies**: US-A02 (주문 대시보드), US-C02 (세션 관리)

#### US-A07: 과거 주문 내역 조회

2. **Given** 과거 주문 목록 **When** 각 주문을 확인하면 **Then** 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
3. **Given** 과거 내역 화면 **When** 날짜 필터를 적용하면 **Then** 해당 날짜 범위의 주문만 표시된다
4. **Given** 과거 내역 화면 **When** "닫기" 버튼을 클릭하면 **Then** 대시보드로 복귀한다
5. **Given** 과거 내역이 없는 테이블 **When** 과거 내역을 조회하면 **Then** "과거 주문 내역이 없습니다" 메시지가 표시된다

**Priority**: Must Have

2. **Given** 메뉴 등록 폼 **When** 필수 필드(메뉴명, 가격, 카테고리)를 입력하고 저장하면 **Then** 메뉴가 등록되고 목록에 표시된다
3. **Given** 메뉴 등록 폼 **When** 필수 필드가 비어있는 상태로 저장을 시도하면 **Then** 유효성 검증 에러가 표시된다
4. **Given** 가격 입력 **When** 0 이하 또는 비정상 값을 입력하면 **Then** 가격 범위 검증 에러가 표시된다
5. **Given** 메뉴 목록 **When** 특정 메뉴의 "수정" 버튼을 클릭하면 **Then** 기존 정보가 채워진 수정 폼이 표시된다
6. **Given** 메뉴 수정 폼 **When** 정보를 변경하고 저장하면 **Then** 메뉴가 업데이트되고 고객 화면에 즉시 반영된다
7. **Given** 메뉴 목록 **When** "삭제" 버튼을 클릭하면 **Then** 확인 팝업 후 메뉴가 삭제된다
8. **Given** 이미지 업로드 **When** 이미지 파일을 선택하면 **Then** 서버에 업로드되고 메뉴에 연결된다

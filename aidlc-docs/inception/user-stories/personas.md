# 테이블오더 서비스 - 사용자 페르소나

---

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 김민수 (대표 페르소나) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~60대 (폭넓은 연령층) |
| **기술 수준** | 초급~중급 (스마트폰/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 내 고정 태블릿 (터치스크린) |
| **사용 빈도** | 매장 방문 시마다 (비정기적) |

### 목표
- 빠르고 쉽게 메뉴를 확인하고 주문하기
- 대기 시간 없이 원하는 메뉴를 선택하기
- 현재 주문 상태를 확인하기

### 동기
- 직원 호출 없이 편하게 주문하고 싶음
- 메뉴 사진과 설명을 보고 신중하게 선택하고 싶음
- 추가 주문을 부담 없이 하고 싶음

### 불만 요소 (Pain Points)
- 복잡한 인터페이스, 작은 버튼
- 주문이 제대로 들어갔는지 불안함
- 이전 손님의 주문 내역이 보이는 것

### 성공 기준
- 3번의 터치 이내로 주문 완료
- 주문 후 즉시 확인 가능
- 별도 로그인/회원가입 불필요

## Persona 2: 매장 관리자 (Store Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 박서연 (대표 페르소나) |
| **역할** | 매장 운영자 / 점장 |
| **연령대** | 30~50대 |
| **기술 수준** | 중급 (PC/태블릿 업무 사용 경험) |
| **사용 환경** | 매장 내 PC 또는 태블릿 (브라우저) |
| **사용 빈도** | 매일 영업 시간 동안 상시 |

### 목표
- 들어오는 주문을 실시간으로 파악하기
- 테이블별 주문 현황을 한눈에 보기
- 메뉴를 쉽게 관리하기 (등록/수정/삭제)
- 테이블 회전율을 효율적으로 관리하기

### 동기
- 주문 누락 방지
- 인건비 절감 (주문 접수 인력 감소)
- 매장 운영 효율성 극대화

### 불만 요소 (Pain Points)
- 주문이 늦게 표시되는 것
- 복잡한 관리 화면
- 잘못된 주문 처리의 어려움

### 성공 기준
- 신규 주문 2초 이내 화면 표시
- 테이블별 현황 한 화면에서 확인
- 메뉴 변경 즉시 고객 화면 반영

## Persona 3: 본사 관리자 (HQ Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 이준호 (대표 페르소나) |
| **역할** | 본사 운영 담당자 |
| **연령대** | 30~40대 |
| **기술 수준** | 중급~고급 (IT 시스템 관리 경험) |
| **사용 환경** | 사무실 PC (브라우저) |
| **사용 빈도** | 주 3~5회 |

### 목표
- 전체 매장 목록을 관리하기
- 새로운 매장을 시스템에 등록하기

### 동기
- 다수 매장의 중앙 관리
- 신규 매장 온보딩 효율화

### 불만 요소 (Pain Points)
- 매장별로 개별 접속해야 하는 번거로움
- 매장 정보 관리의 비효율

### 성공 기준
- 전체 매장 현황 한눈에 파악
- 신규 매장 등록 5분 이내 완료

### MVP 범위 참고
- MVP에서는 매장 등록/조회만 포함 (최소 기능)

## 페르소나 간 관계

```
+------------------+          +---------------------+
|   본사 관리자     |  등록    |    매장 (Store)      |
|   (HQ Admin)    |--------->|                     |
+------------------+          +---------------------+
                                       |
                              관리/운영  |
                                       v
                              +---------------------+
                              |   매장 관리자         |
                              |   (Store Admin)     |
                              +---------------------+
                                       |
                         테이블 설정/    |  주문 모니터링
                         세션 관리      |
                                       v
                              +---------------------+
                              |     고객             |
                              |   (Customer)        |
                              +---------------------+
```

### 상호작용 흐름
1. **본사 관리자** → 매장 등록 → **매장 관리자** 계정 생성 가능
2. **매장 관리자** → 테이블 초기 설정 → **고객** 자동 로그인 활성화
3. **고객** → 주문 생성 → **매장 관리자** 실시간 수신
4. **매장 관리자** → 주문 상태 변경 → **고객** 상태 확인 (폴링)
5. **매장 관리자** → 이용 완료 처리 → 세션 종료 → 다음 **고객** 새 세션 시작 가능

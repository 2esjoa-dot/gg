## Unit 1: Backend API — 스토리 매핑

모든 유저 스토리는 백엔드 API 구현을 필요로 합니다.

| Story | 백엔드 구현 내용 |
|---|---|
| US-C01 | 태블릿 인증 API, 토큰 발급 |
| US-C02 | 세션 생성/조회 로직 |
| US-C03 | 메뉴 조회 API (카테고리별) |
| US-C04 | — (프론트 전용, 로컬 스토리지) |
| US-C05 | 주문 생성 API, 세션 연동 |
| US-C06 | 세션별 주문 조회 API |
| US-A01 | 관리자 인증 API, JWT, 로그인 제한 |
| US-A02 | SSE 스트림 API, 주문 이벤트 발행 |
| US-A03 | 주문 상태 변경 API |
| US-A04 | 테이블 등록 API |
| US-A05 | 주문 삭제 API, 총액 재계산 |
| US-A06 | 이용 완료 API, 세션 종료, 이력 이동 |
| US-A07 | 과거 주문 내역 조회 API |
| US-A08 | 메뉴 CRUD API, 이미지 업로드 |
| US-A09 | 메뉴 순서 변경 API |
| US-A10 | 관리자 계정 등록 API |
| US-H01 | 매장 등록 API |
| US-H02 | 매장 목록 조회 API |

## Unit 2: Customer Frontend — 스토리 매핑

| Story | 프론트엔드 구현 내용 |
|---|---|
| US-C01 | 자동 로그인 로직, 초기 설정 페이지, 로컬 스토리지 인증 정보 |
| US-C02 | 세션 상태 표시 (UI 레벨, 백엔드 연동) |
| US-C03 | 메뉴 페이지, 카테고리 탭, 메뉴 카드, 상세 모달 |
| US-C04 | 장바구니 UI, 수량 조절, 총액 계산, 로컬 스토리지 동기화 |
| US-C05 | 주문 확인 페이지, 주문 성공/실패 처리, 5초 리다이렉트 |
| US-C06 | 주문 내역 페이지, 30초 폴링, 상태 배지 |

## 크로스 유닛 스토리

| Story | 관련 Unit | 설명 |
|---|---|---|
| US-C05 + US-A02 | Unit 1 + Unit 2 + Unit 3 | 고객 주문 → 백엔드 처리 → SSE로 관리자 수신 |
| US-A03 + US-C06 | Unit 1 + Unit 3 + Unit 2 | 관리자 상태 변경 → 백엔드 저장 → 고객 폴링으로 확인 |
| US-A06 + US-C02 | Unit 1 + Unit 3 + Unit 2 | 관리자 이용 완료 → 세션 종료 → 다음 고객 새 세션 |
